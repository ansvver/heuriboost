# Reckless Input Builder

## Overview

Convert user-owned RAG materials into the two user-authored files expected by
the HeuriBoost production-case repair flow:

- `base_dataset.csv` or `base_dataset.jsonl`
- `production_cases.csv` or `production_cases.jsonl`

Do not ask users to hand-author `.heuriboost/compiled/`,
`query_doc_examples.csv`, `regression_cases.yaml`, or `case_sets/`. Those are
generated by the sibling `$heuriboost-rag` skill's `scripts/compile_cases.py`.

Read [reckless-input-contract.md](references/reckless-input-contract.md) before
writing files or judging whether the user's materials are sufficient.

## Runtime Path

Resolve the sibling HeuriBoost runtime relative to this skill directory:

```bash
RECKLESS_INPUT_SKILL_DIR="<absolute path to this skill directory>"
HEURIBOOST_RAG_SKILL_DIR="$(cd "$RECKLESS_INPUT_SKILL_DIR/../heuriboost-rag" && pwd)"
```

## Workflow

1. Inventory the material.
   Identify query text, retrieved document text, document ids, ranks, scores,
   user verdicts, relevance labels, domains, timestamps, and existing split
   assignments. Keep source ids when available.

2. Decide whether full or weak repair is possible.
   Full repair needs at least one `good` target for each actionable production
   case. Bad-only cases can be encoded, but they require
   `--acceptance-level weak` and are not promotion eligible.

3. Build `base_dataset`.
   This is normal query-document training and evaluation evidence, not the
   current production failure rows. Prefer CSV for flat tables. Use JSONL when
   the source has nested candidate arrays. If no trustworthy split exists,
   omit `split` and let `compile_cases.py` assign deterministic query-group
   splits.

4. Build `production_cases`.
   Each row is one shown document for a current production repair case. Rows
   sharing a `case_id` must share the same `domain` and `query`. Use
   `user_verdict=good` for the desired document, `bad` for the document to
   suppress, and `unknown` only for extra context candidates.

5. Validate by compiling.

   ```bash
   python3 "$HEURIBOOST_RAG_SKILL_DIR/scripts/compile_cases.py" \
     --base-dataset path/to/base_dataset.csv \
     --production-cases path/to/production_cases.csv \
     --output-dir path/to/repair_output \
     --strict
   ```

6. If compile succeeds, tell the user the reckless repair command.

   ```bash
   python3 "$HEURIBOOST_RAG_SKILL_DIR/scripts/repair_reranker.py" \
     --base-dataset path/to/base_dataset.csv \
     --production-cases path/to/production_cases.csv \
     --output-dir path/to/repair_output \
     --reckless \
     --acceptance-level full
   ```

## Output Rules

- Prefer writing files with Python `csv`/`json` modules or pandas, not manual
  comma joining.
- Normalize whitespace in free text, but preserve source meaning.
- Never invent a `good`, `bad`, or relevance label. If evidence is ambiguous,
  either mark production context as `unknown` or ask for the missing judgment.
- Keep generated files separate from raw user material.
- Summarize created file paths, row counts, touched domains, compile warnings,
  and the exact validation command that ran.

## Quick Examples

Minimal `base_dataset.csv`:

```csv
domain,query,text,relevance,split
tax,How do I deduct home office expenses?,Home office deduction rules require exclusive business use.,good,train
tax,How do I deduct home office expenses?,Corporate office lease accounting rules for headquarters.,bad,train
```

Minimal full `production_cases.csv`:

```csv
domain,case_id,query,shown_doc_text,user_verdict,rank
tax,prod_home_office,How do I deduct home office expenses?,Corporate office lease accounting rules for headquarters.,bad,1
tax,prod_home_office,How do I deduct home office expenses?,Home office deduction rules require exclusive business use.,good,5
```

Known fixtures: `examples/fiqa/repair/base_dataset_minimal.csv`,
`examples/fiqa/repair/production_cases_full.csv`, and
`examples/fiqa/repair/production_cases_bad_only.csv`.
